Introduction

What is CSS?

Cascading Style Sheets, commonly known as CSS, is a fundamental technology used in web development to control the presentation and formatting of web documents. In simpler terms, CSS is responsible for how web content looks and is displayed on a user's screen. CSS works in tandem with HTML (Hypertext Markup Language) and JavaScript to build interactive and visually appealing websites. While HTML defines the structure and content of a web page, CSS takes care of its appearance, including layout, colors, fonts, and spacing. CSS is essential for separating content from design, making websites more maintainable and adaptable. By creating reusable stylesheets, web developers can ensure consistency across a website and quickly update the look and feel of the entire site.

A Brief History of CSS

The history of CSS dates back to the early days of the World Wide Web. Prior to the introduction of CSS, web developers faced significant challenges in controlling the appearance of web pages. Styles were often applied directly within HTML elements using attributes like bgcolor, text, and align, leading to messy and hard-to-maintain code. In 1996, the World Wide Web Consortium (W3C) released CSS1 as a formal specification. CSS1 provided a structured way to describe the visual presentation of web documents. However, early browser support was limited, and developers encountered compatibility issues. CSS2, introduced in 1998, brought significant improvements and extended the capabilities of CSS. It introduced features like positioning, floats, and more advanced selectors. This version marked a major step forward in web design and layout possibilities. CSS2.1, published in 2004, focused on clarifying the specifications and improving browser support for existing features. CSS3, introduced in the mid-2000s, is not a single specification but a collection of modules, each adding new capabilities to CSS. This modular approach allows for gradual adoption of new features without waiting for a complete CSS4 specification. CSS3 introduced transformations, animations, gradients, and much more, expanding the creative possibilities for web designers.

Importance of CSS in Web Development

CSS plays a pivotal role in web development for several reasons:

  • Separation of Concerns: One of the core principles of web development is the separation of concerns. CSS allows for the separation of content (HTML), structure (HTML), and presentation (CSS). This separation makes web development more organized and manageable. Changes to the design can be made without altering the underlying content.
  • Consistency: CSS enables the creation of consistent styles across an entire website. By defining styles in a centralized manner, developers ensure that headings, paragraphs, links, and other elements have a consistent appearance.
  • Maintainability: As websites grow in complexity, maintaining a consistent look and feel becomes challenging. CSS simplifies this task by providing a single source of truth for styles. Changes can be applied universally by updating the CSS rules, reducing maintenance overhead.
  • Adaptability: With the proliferation of devices and screen sizes, responsive web design is essential. CSS, along with media queries, allows developers to create flexible layouts that adapt gracefully to different screen dimensions, making websites accessible on various devices, from smartphones to large desktop monitors.
  • Accessibility: CSS can be used to improve the accessibility of web content. Developers can use CSS to provide proper contrast, define text sizes, and create layouts that work well with screen readers, ensuring that websites are accessible to users with disabilities.

Core Concepts of CSS

Before diving into the details of CSS, it's essential to understand some fundamental concepts that underlie its structure and functionality.

Selectors

At the heart of CSS are selectors, which are patterns used to select the HTML elements you want to style. Selectors define the target elements based on their type, attributes, relationships, or other criteria.

  • Type Selectors: These select elements by their HTML tag name. For example, to select all

    elements, you would use the selector p.

  • Class selector: Class selectors target elements with a specific class attribute value. For example, to select all elements with the class "button," you would use the selector button. .center { text-align: center; color: red; }
  • ID Selectors: ID selectors target a single element with a unique id attribute value. For example, to select an element with the id "header," you would use the selector #header. #firstname { background-color: yellow; }
  • Attribute Selectors: These select elements based on their attributes and attribute values. For instance, [data-type="quote"] would select elements with a data-type attribute set to "quote." input[type="text"] { width: 150px; display: block; margin-bottom: 10px; background-color: yellow; } input[type="button"] { width: 120px; margin-left: 35px; display: block; }

Properties and Values

CSS properties define the specific style aspects you want to apply to selected elements, while values determine the exact styling for each property. There is a wide range of CSS properties, each responsible for different visual aspects of web elements.

  • Text Properties: color: Specifies the text color (e.g., red, #00ff00, rgb(255, 0, 0)). font-family: Sets the font family for text (e.g., "Arial", sans-serif). font-size: Determines the size of text (e.g., 16px, 1.2em).

Box Model

The CSS Box Model is a fundamental concept that defines how elements are rendered in web documents. According to the Box Model, every HTML element is treated as a rectangular box with four components:

  • Content: actual content of the element, such as text or images.
  • Padding: The space between the content and the element's border.
  • Border: border surrounding the padding and content.
  • Margin: The space between the border and adjacent elements.

CSS Selectors

CSS selectors are at the heart of applying styles to HTML elements. Selectors define the criteria for targeting specific elements, allowing you to apply CSS rules to those elements.

Type selectors

  • Type selectors, also known as tag selectors, target elements based on their HTML tag names. For example, if you want to select all

    elements on a page to change their font size, you can use the type selector p: p { font-size: 16px; }

Class Selectors

Class selectors are used to target elements with a specific class attribute value. They are preceded by a dot (.) followed by the class name. For instance, to select all elements with the class "button" and give them a background color, you can use the following class selector:

.button { background-color: #ff9900; }

In your HTML, you would apply the "button" class to elements like this:

button class="button">Click me button>

Class selectors are versatile and allow you to style multiple elements with the same class across your website.

ID Selectors

ID selectors are used to target a single HTML element with a unique id attribute value. They are preceded by a hash (#) followed by the id name. Here's an example:

#header { font-size: 24px; }

n this case, the CSS rule sets the font size to 24 pixels for the element with the id "header." IDs must be unique within a document, so this selector targets only one specific element.

Attribute Selectors

Attribute selectors allow you to target elements based on their attributes and attribute values. You can use square brackets ([]) to enclose the attribute selector and specify the attribute and its value. For example

[data-type="quote"] { font-style: italic; }

In this rule, the font-style property is applied to elements with a data-type attribute set to "quote."

This is a quoted text.

Pseudo-classes and Pseudo-elements

Pseudo-classes and pseudo-elements are selectors that target elements based on their state or position within the document.

Pseudo-classes

Pseudo-classes start with a colon (:) and allow you to select elements based on user interactions and states. Some common pseudo-classes include:

  • :hover: an element when the user hovers the mouse pointer over it. It's commonly used for interactive elements like links and buttons. a:hover { text-decoration: underline; }
  • :active: Selects an element while it is being activated (clicked). It's often used for styling buttons to provide feedback when they are clicked. .button:active { background-color: #ff5500; }
  • :focus: Selects an element when it gains focus, typically through keyboard navigation. Useful for styling form inputs. input:focus { border-color: #007bff; }

What is The Viewport?

The viewport is the user's visible area of a web page.

The viewport varies with the device, and will be smaller on a mobile phone than on a computer screen.

Before tablets and mobile phones, web pages were designed only for computer screens, and it was common for web pages to have a static design and a fixed size.

Then, when we started surfing the internet using tablets and mobile phones, fixed size web pages were too large to fit the viewport. To fix this, browsers on those devices scaled down the entire web page to fit the screen.

Setting The Viewport

HTML5 introduced a method to let web designers take control over the viewport, through the <meta> tag.

You should include the following <meta> viewport element in all your web pages:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This gives the browser instructions on how to control the page's dimensions and scaling.

The width=device-width part sets the width of the page to follow the screen-width of the device (which will vary depending on the device).

The initial-scale=1.0 sets the initial zoom level when the page is first loaded by the browser

Size Content to The Viewport

Users are used to scroll websites vertically on both desktop and mobile devices - but not horizontally!
So, if the user is forced to scroll horizontally, or zoom out, to see the whole web page it results in a poor user experience.

Some additional rules to follow:

  1. Do NOT use large fixed width elements - For example, if an image is displayed at a width wider than the viewport it can cause the viewport to scroll horizontally. Remember to adjust this content to fit within the width of the viewport.
  2. Do NOT let the content rely on a particular viewport width to render well - Since screen dimensions and width in CSS pixels vary widely between devices, content should not rely on a particular viewport width to render well.
  3. Use CSS media queries to apply different styling for small and large screens - Setting large absolute CSS widths for page elements will cause the element to be too wide for the viewport on a smaller device. Instead, consider using relative width values, such as width: 100%. Also, be careful of using large absolute positioning values. It may cause the element to fall outside the viewport on small devices.

What is a Media Query?

Media query is a CSS technique introduced in CSS3. It uses the @media rule to include a block of CSS properties only if a certain condition is true.

@media only screen and (max-width: 600px) { body { background-color: lightblue; } }

CSS Animations:

What are CSS Animations?

An animation lets an element gradually change from one style to another.

You can change as many CSS properties you want, as many times as you want.

To use CSS animation, you must first specify some keyframes for the animation.

Keyframes hold what styles the element will have at certain times.

The @keyframes Rule

When you specify CSS styles inside the @keyframes rule, the animation will gradually change from the current style to the new style at certain times. To get an animation to work, you must bind the animation to an element. The following example binds the "example" animation to the element. The animation will last for 4 seconds, and it will gradually change the background-color of the element from "red" to "yellow":

Example

/* The animation code */ @keyframes example { from {background-color: red;} to {background-color: yellow;} } /* The element to apply the animation to */ div { width: 100px; height: 100px; background-color: red; animation-name: example; animation-duration: 4s; } click it

Note: The animation-duration property defines how long an animation should take to complete. If the animation-duration property is not specified, no animation will occur, because the default value is 0s (0 seconds).

In the example above we have specified when the style will change by using the keywords from and to (which represents 0% (start) and 100% (complete)).

It is also possible to use percent. By using percent, you can add as many style changes as you like

Example

/* The animation code */ @keyframes example { 0% {background-color: red;} 25% {background-color: yellow;} 50% {background-color: blue;} 100% {background-color: green;} } /* The element to apply the animation to */ div { width: 100px; height: 100px; background-color: red; animation-name: example; animation-duration: 4s; } click it

Delay an Animation

The animation-delay specifies a delay for the start of an animation. The following example has a 2 seconds delay before starting the animation:

Example.

div { width: 100px; height: 100px; position: relative; background-color: red; animation-name: example; animation-duration: 4s; animation-delay: 2s; } click it

Negative values are also allowed. If using negative values, the animation will start as if it had already been playing for N seconds. In the following example, the animation will start as if it had already been playing for 2 seconds:

Example

div { width: 100px; height: 100px; position: relative; background-color: red; animation-name: example; animation-duration: 4s; animation-delay: -2s; } click it

Set How Many Times an Animation Should Run

The animation-iteration-count property specifies the number of times an animation should run. The following example will run the animation 3 times before it stops:

Example

div { width: 100px; height: 100px; position: relative; background-color: red; animation-name: example; animation-duration: 4s; animation-iteration-count: 3; } click it

The following example uses the value "infinite" to make the animation continue for ever:

Example

div { width: 100px; height: 100px; position: relative; background-color: red; animation-name: example; animation-duration: 4s; animation-iteration-count: infinite; } click it

CSS selectors are used to select the content you want to style. Selectors are the part of CSS rule set. CSS selectors select HTML elements according to its id, class, type, attribute etc.

There are several different types of selectors in CSS.

  1. css element selector
  2. css id selectors
  3. css class selector
  4. css universal selector
  5. css group selector

CSS Element Selector

The element selector selects the HTML element by name.

h2, p { background-color: yellow; } click it

css id selectors

The id selector uses the id attribute of an HTML element to select a specific element. The id of an element is unique within a page, so the id selector is used to select one unique element! To select an element with a specific id, write a hash (#) character, followed by the id of the element.

Example

#para1 { text-align: center; color: red; } click it

The CSS class Selector

The class selector selects HTML elements with a specific class attribute. To select elements with a specific class, write a period (.) character, followed by the class name.

Example

.center { text-align: center; color: red; } click it

The CSS Universal Selector

he universal selector (*) selects all HTML elements on the page.

Example

* { text-align: center; color: blue; } click it

The CSS Grouping Selector

The grouping selector selects all the HTML elements with the same style definitions. Look at the following CSS code (the h1, h2, and p elements have the same style definitions)

h1 { text-align: center; color: red; } h2 { text-align: center; color: red; } p { text-align: center; color: red; }

It will be better to group the selectors, to minimize the code.

To group selectors, separate each selector with a comma.

Example

h1, h2, p { text-align: center; color: red; } click it
